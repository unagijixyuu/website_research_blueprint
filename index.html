🧭 全端學習建議順序圖（含 Figma, Vue, FastAPI, Django, MySQL）

🔰 階段一：前端基礎打底（你已完成 🎉）

  • ✅ HTML / CSS / JavaScript
  • 🎯 目標：能寫靜態網頁，了解基本 DOM 操作與事件

🎨 階段二：UI/UX 設計 & Figma（快速進行）

  • 學習 Figma 基本操作、畫 wireframe、組元件、設計排版
  • 🎯 目標：畫出租車網站的「首頁、車輛頁、預約頁」設計草稿

👉 不用太深入 Figma，學會畫 + export 即可，之後還會一直回來補設計的

⚙️ 階段三：Vue 前端框架學習（功能整合開始）

  • 學習元件化、props、emit、v-bind、v-model
  • 學會 Vue Router（單頁應用）、基本 Vue 生命週期
  • 🎯 目標：用 Vue 將 Figma 畫的畫面做出來 + 加一點假資料跑動畫互動

🛠️ 階段四：MySQL 資料庫設計與基礎操作

  • ER 模型、表格設計、資料型別、JOIN、CRUD 查詢
  • 🎯 目標：規劃租車系統的資料表（例如：用戶、車輛、訂單）

👉 建議這階段只先用「圖形化工具」設計資料結構，方便接後端用

🌐 階段五：Django 框架（資料處理的主力核心）

Django 是你系統的核心：處理會員登入、資料 CRUD、後台管理
  • 學習 MVT 架構、Model 設計、View 撰寫、Admin 管理、API 串接（Django REST Framework）
  • 🎯 目標：做出租車網站的 API，例如 /api/cars, /api/orders, /api/login

⚡ 階段六：FastAPI（高速外部服務或串接用途）

FastAPI 主要用於：外部資料串接（像 TeslaMate）、非同步處理、快速提供特定功能
  • 學習路由、Request/Response、Pydantic、async/await
  • 🎯 目標：做一個小型外部 API 服務（例如：接收 Tesla API 回傳數據，並轉存至 Django）

👉 FastAPI 跟 Django 是互補關係，不是互相取代
